Day 3 · 50 XP

Mission: AI Can Be Wrong

Learn why AI can sound confident and still be wrong, then create a simple trust checklist before using AI answers.

Time

55 min

Keep it focused and practical.

AI Skill

Verification

One skill, used immediately.

Output

Trust checklist

Something visible by the end.

Proof

Screenshot + log

Evidence of progress.

Mission timeline

0–10

Confidence trap

Ask AI a factual question and notice how confident it sounds.

10–25

Check the answer

Identify which parts need verification and which are safe general advice.

25–40

Build checklist

Create red/yellow/green rules for trusting AI output.

40–50

Test checklist

Run the checklist on one school question and one LEGO/business question.

50–55

XP log

Write one rule Sahil will follow before trusting AI.

Starter prompt

Review this AI answer: [paste answer]. Break it into claims. Mark each claim green if safe, yellow if needs checking, and red if risky or unsupported. Explain why.

Done means done

  • Trust checklist exists.
  • One screenshot or note is saved.
  • One thing that worked is recorded.
  • One thing to improve is recorded.
  • 50 XP is logged.
