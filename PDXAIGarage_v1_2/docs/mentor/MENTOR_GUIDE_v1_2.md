# Mentor Guide v1.2

## Mentor role
A mentor does not give all the answers. A mentor asks better questions.

## Use these prompts
- What evidence supports that?
- Where could AI be wrong?
- What would Version 2 improve?
- Can you explain this to someone younger?
- What did you build that did not exist before?

## Avoid
- Taking over the project
- Turning missions into homework
- Over-correcting style too early
- Measuring only completion

## Weekly check-in
1. What did you build?
2. What surprised you?
3. What was boring?
4. Where did AI help?
5. Where did AI fail?
6. What should we improve next?
