Portal QA · v6.1 QA check

Testing checklist before we keep building.

Use this page to test the site like a real user. The goal is to catch broken links, confusing pages, mobile issues, and places where the experience still feels too much like a document.

Core pages to test

  • Home loads and Start Here button works.
  • AI Learning Lab opens and links to all learning labs.
  • Workspace saves a mission log locally.
  • Artifact Vault displays saved artifacts.
  • Badge Board updates after mission logs are saved.
  • Start Here links to Day 1 and Showcase.
  • Dashboard mission cards open the correct mission pages.
  • School Tools page opens and links Day 11–15.
  • Sports Module opens and links Day 16–20.
  • Mini Apps page opens and links Day 21–25.
  • Final Sprint page opens and links Day 26–30.
  • Curriculum shows the six-week path.
  • Lesson Plans includes 30 sessions and quick launch links.
  • Projects links to LEGO Case File.
  • Prompt Library and Resources are readable on mobile.
  • Mobile hamburger opens the navigation menu.
  • Showcase roadmap makes sense.

Things to watch

  • Navigation may get crowded on smaller screens.
  • Dashboard XP is currently static, not saved to a database.
  • Mission logs are templates, not forms yet.
  • HTTPS may still be finalizing if GitHub certificate is pending.
  • Some later weeks are planned but not detailed yet.
